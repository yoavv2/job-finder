<objective>
Establish the resume **source of truth as structured data** (RES-01): a Zod `ResumeSchema` covering Profile, Summary, Skills, Experience, Projects, Education, Certificates, Languages, plus a `loadResume(path)` that reads `resume/master.yaml`, validates against the schema, and fails fast with a clear error.

This implements the locked decision: the resume lives as structured data; **PDFs are OUTPUTS ONLY and are NEVER parsed as input.** This module reads YAML, never PDF. The schema + loader are the substrate Plan 05 (renderer + validator) and Phase 4 (tailoring) build on — tailoring will be structured-in/structured-out against this exact schema.

Mirror the Phase 1 config pattern (single Zod schema as source of truth, `z.infer` type, fail-fast `loadConfig`-style loader using the `yaml` package already in deps).

Purpose: A typed, validated resume model that everything downstream renders/validates against.
Output: ResumeSchema + Resume type, loadResume loader, and a complete example master.yaml.
</objective>
