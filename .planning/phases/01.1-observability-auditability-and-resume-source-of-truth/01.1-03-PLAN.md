<objective>
Build the central run-history framework: a `runAgent(agent, ctx)` wrapper that automatically records one `AgentRuns` row per execution (OBS-02), driving the STARTED -> RUNNING -> SUCCESS/FAILED lifecycle, and aggregating token usage + estimated cost accrued through the LLM provider during that run (OBS-03).

This implements the locked decision: **the framework emits Run records centrally — individual agents MUST NOT create them.** Agents stay unchanged; callers invoke `runAgent(agent, ctx)` instead of `agent.run(ctx)` directly. A thrown error still finalizes the run as FAILED with the error captured, then re-throws.

Token aggregation mechanism (the "clean mechanism" the decision asks us to decide + document): a per-run `UsageAccumulator` exposed to the run, and an `LLMProvider` decorator (`wrapWithUsageTracking`) that adds every call's `Usage` to the accumulator. `runAgent` wraps `ctx.llm` with the accumulator before invoking the agent, then reads `tokens`/`estimatedCost` from the accumulator when finalizing — so any LLM call the agent makes is counted without the agent knowing.

Purpose: Observability that is automatic and impossible for an agent to forget.
Output: AgentRunRepository, UsageAccumulator, usage-tracking provider decorator, and the runAgent wrapper.
</objective>
