<objective>
Add the three historical-data tables — `agentRuns`, `jobEvents`, `artifacts` — to the Drizzle schema and generate the migration that creates them. This is the schema foundation for the entire observability/audit layer; the run-history wrapper (Plan 03) and the event/artifact repositories (Plan 02) build directly on these typed defs.

This plan enforces the locked architectural boundary: current-state tables (companies/jobs/applications) are NOT touched. These three new tables are pure historical data ("what happened over time"). `jobEvents` is shaped append-only (no `updatedAt` column — it is never mutated). `artifacts.type` is free-form text and `metadata` is JSON so new artifact types never require a schema change.

Purpose: Every downstream observability/audit plan needs these tables + inferred types to exist first.
Output: Updated `src/db/schema.ts` with three new tables + inferred types, and a generated `0001_*` migration applied on DB open.
</objective>
