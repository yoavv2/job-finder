<objective>
Build the repository layer over the `jobEvents` and `artifacts` tables created in Plan 01, and wire them into `buildRepositories`. All event and artifact access goes through these repos — no raw SQL or Drizzle imports leak into agents (the locked Phase 1 repository-pattern rule).

`JobEventRepository` is the append-only audit-trail seam (EVT-02/EVT-03): it exposes `append` + read-by-job and deliberately has NO update/delete method, so history can never be overwritten. `Job.status` remains the current-state field elsewhere; this repo is the immutable history.

`ArtifactRepository` (ART-02/ART-03) is how Jobs reference artifacts — via the generic table, never per-type path columns. It offers list-by-job and list-by-job-and-type helpers.

Purpose: Give Phase 2-4 agents (and Plan 03's run wrapper) a clean, ORM-free way to record events and artifacts.
Output: Two repository classes + their tests, and an updated `buildRepositories` barrel.
</objective>

A repository takes the Drizzle db instance: `type DrizzleDb = Database['db']` and `constructor(private readonly db: DrizzleDb) {}`. Use Drizzle query builder (`.insert(...).values(...).returning().get()`, `.select().from(...).where(eq(...)).all()`) — NEVER raw SQL outside src/db.

Existing barrel (src/db/repositories/index.ts):
```typescript
export interface Repositories { jobs: JobRepository; companies: CompanyRepository; applications: ApplicationRepository; }
export function buildRepositories(handles: Database): Repositories {
  return {
    jobs: new JobRepository(handles.db, handles.sqlite),
    companies: new CompanyRepository(handles.db),
    applications: new ApplicationRepository(handles.db),
  };
}
```

Inferred types available from schema (Plan 01): JobEvent/NewJobEvent, Artifact/NewArtifact.
Clock: callers pass timestamps via the injected `now()` clock where deterministic tests matter; createdAt has a SQL default, but accept an optional `createdAt?: Date` on inputs so the run wrapper / agents can pass the injected clock.
</interfaces>
